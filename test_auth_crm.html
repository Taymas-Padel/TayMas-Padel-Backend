🧪 Тест-план — сценарии проверки
Пошаговые инструкции для полной проверки системы.
Сценарий 1 — Новый клиент
1. 📲 Отправить код → любой новый номер
2. 🔑 Ввести код + device_id → получить токены
3. ✅ Проверить: is_new_user=true, is_profile_complete=false
4. 👤 Заполнить имя/фамилию
5. 🪪 Загрузить профиль → is_profile_complete=true
6. 🚪 Выйти
Сценарий 2 — Повторный вход + смена устройства
1. 📲 Отправить код на существующий номер
2. 🔑 Ввести код с ДРУГИМ device_id
3. ✅ Проверить: is_qr_blocked=true
4. 🏢 Войти в CRM как ресепшн (по паролю)
5. 🔍 Найти клиента по номеру
6. ✅ Разблокировать QR
7. 🪪 Проверить профиль клиента → is_qr_blocked=false
Сценарий 3 — Разделение доступа
1. Создай в Django Admin пользователя role=RECEPTIONIST с паролем
2. 📲 Попробуй SMS-вход с его номером → ❌ ошибка 403
3. 🏢 Войди по паролю через CRM → ✅ успех
4. 📲 Попробуй CRM-вход для клиента → ❌ ошибка 403
Сценарий 4 — Rate-limit
1. 📲 Отправить код 4 раза подряд → на 4-й получить 429
2. 🔑 Ввести неверный код 6 раз → блокировка номера на 10 мин
