PROJECT CONTEXT: Padel Center CRM & Mobile API

1. Описание системы Разработка комплексной системы управления падел-центром. Система состоит из единого бэкенда (Django), который обслуживает два фронта:

CRM (Web): Интерфейс для администраторов, менеджеров и тренеров (управление бронированием, клиентами, финансами).

Mobile App API: Бэкенд для нативного мобильного приложения клиентов (бронирование, профиль, геймификация).

2. Технический стек

Language: Python 3.x

Framework: Django 5.x (Core Logic + Admin Interface)

API: Django REST Framework (DRF) — для мобильного приложения.

Database: PostgreSQL (Единая база данных для CRM и Мобилки).

Tasks: Celery + Redis (для фоновых задач: уведомления, отчеты).

Integrations: OpenAI API (аналитика), Telegram Bot (отчеты), Payment Gateway.

3. Архитектура

Monolith: Единый проект Django.

Single Source of Truth: База данных одна. Логика проверки доступности кортов, расчета цены и правил отмены находится строго в Django. Мобильное приложение общается только через API.

CRM UI: Рендеринг через Django Templates (DTL) или Django Admin (кастомизированная).

4. Ключевые Сущности и Модели (Database Schema)

Users & Roles: Кастомная модель пользователя. Роли: SuperAdmin, Manager, Coach_Padel, Coach_Fitness, Client.

Courts (Корты): Номер, тип покрытия, статус (Active/Maintenance), базовые цены.

Schedule & Slots: Сетка расписания. Логика предотвращения овербукинга (Double Booking).

Bookings (Брони): Связь Client + Court + TimeSlot + Trainer (опц.) + Services (инвентарь). Статусы: Pending, Confirmed, Canceled, Completed.

Subscriptions (Абонементы): Типы абонементов, баланс занятий, срок действия, заморозка.

Matchmaking (Lobby): Сущность "Игровая комната" для поиска партнеров (уровни игры: Novice, Pro).

Gamification: Рейтинг ELO, Лиги, Достижения (Achievements), История матчей.

5. Основной функционал (Modules)

Модуль Бронирования: Шахматка расписания (Calendar View), динамическое ценообразование (вечер/выходные дороже), валидация времени.

CRM / Sales: Воронка продаж (Leads Kanban), база клиентов, история взаимодействий.

Финансы: Учет транзакций, дебиторская задолженность, интеграция с кассой/эквайрингом.

Тренерский кабинет: Личное расписание, список учеников, учет групповых и индив. тренировок.

AI Analytics: Еженедельный сбор данных -> OpenAI API -> Аналитический отчет в Telegram владельца.

Задача текущего этапа: Проектирование архитектуры БД (models.py) и настройка базового конфига.
