# Бриф для фронта: тариф тренера по числу игроков (1–2 и 3–4)

Бэкенд поддерживает **два тарифа тренера за час** в зависимости от количества игроков (по документу «Падел-тренировки с тренером»):

- **1–2 игрока** — своя цена за час (например 24 000 ₸)
- **3–4 игрока** — своя цена за час (например 36 000 ₸)

Если эти поля не заданы, используется общая **Цена за час (по умолчанию)** — `price_per_hour`.

---

## 1. Где отображать и что отправлять

### 1.1 Список тренеров (формы брони, лобби)

**GET /api/auth/coaches/**

В ответе у каждого тренера теперь есть (в дополнение к существующим полям):

| Поле | Тип | Описание |
|------|-----|----------|
| `coach_price` | number | Цена по умолчанию (как раньше, из `price_per_hour`) |
| `coach_price_1_2` | number \| null | Цена за час для 1–2 игроков |
| `coach_price_3_4` | number \| null | Цена за час для 3–4 игроков |

**Для превью стоимости тренера в форме брони/лобби:**  
Используйте число игроков (1+друзья в обычной брони, или кол-во участников в лобби):

- 1–2 человека → брать `coach_price_1_2 ?? coach_price`
- 3–4 человека → брать `coach_price_3_4 ?? coach_price`

В CRM в блоке «Новая бронь» при выборе тренера можно показывать, например:  
«24 000 ₸/ч (1–2 чел.) · 36 000 ₸/ч (3–4 чел.)» или только одну строку, в зависимости от выбранного числа участников.

---

### 1.2 Создание сотрудника (в т.ч. тренера)

**POST /api/auth/staff/**

В теле запроса можно передавать (для тренеров имеет смысл заполнять):

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `price_per_hour` | number | нет | Цена за час по умолчанию (как раньше) |
| `coach_price_1_2` | number \| null | нет | Цена тренера за час при 1–2 игроках |
| `coach_price_3_4` | number \| null | нет | Цена тренера за час при 3–4 игроках |

Пример для тренера:

```json
{
  "username": "coach1",
  "first_name": "Алексей",
  "last_name": "Тренер",
  "phone_number": "+77001112233",
  "role": "COACH_PADEL",
  "price_per_hour": 24000,
  "coach_price_1_2": 24000,
  "coach_price_3_4": 36000,
  "password": "...",
  "password_confirm": "..."
}
```

---

### 1.3 Редактирование сотрудника (в т.ч. тренера)

**PATCH /api/auth/staff/<id>/**

В теле можно передавать те же поля (только те, что меняются):

| Поле | Описание |
|------|----------|
| `price_per_hour` | Цена за час по умолчанию |
| `coach_price_1_2` | Цена за час при 1–2 игроках (можно отправить `null`, чтобы сбросить) |
| `coach_price_3_4` | Цена за час при 3–4 игроках (можно отправить `null`, чтобы сбросить) |

Пример:

```json
{
  "coach_price_1_2": 24000,
  "coach_price_3_4": 36000
}
```

---

### 1.4 Детали сотрудника / список сотрудников

**GET /api/auth/staff/** и **GET /api/auth/staff/<id>/**

В объекте сотрудника в ответе теперь есть:

- `price_per_hour` — как раньше  
- `coach_price_1_2` — number или null  
- `coach_price_3_4` — number или null  

Их нужно показывать в карточке/форме тренера и в форме редактирования.

---

## 2. Что сделать в CRM

1. **Создание сотрудника (форма «Добавить сотрудника»)**  
   Если роль = тренер (COACH_PADEL / COACH_FITNESS), показать три поля:
   - Цена за час (по умолчанию)
   - Цена за час (1–2 игрока) — необязательно
   - Цена за час (3–4 игрока) — необязательно  
   При отправке **POST /api/auth/staff/** передавать в body эти три поля (числа или не передавать / null).

2. **Редактирование сотрудника**  
   В форме редактирования для тренера добавить те же три поля; при сохранении отправлять в **PATCH /api/auth/staff/<id>/** только изменённые (в т.ч. `coach_price_1_2`, `coach_price_3_4`).

3. **Карточка/список тренеров**  
   При отображении тренера можно показывать, например:  
   «24 000 ₸/ч (1–2) · 36 000 ₸/ч (3–4)» если заданы оба тарифа, иначе одну цену (по умолчанию или один из тарифов).

4. **Форма новой брони (выбор тренера)**  
   В селекте тренеров или в превью суммы: при известном числе участников (1+друзья или участники лобби) считать стоимость тренера по правилу: 1–2 чел. → `coach_price_1_2 ?? coach_price`, 3–4 чел. → `coach_price_3_4 ?? coach_price`. Бэкенд уже считает итог по этому правилу при создании брони и лобби; на фронте нужно только корректно показывать превью и подставлять нужную цену в подписи.

---

## 3. Кратко

- Во всех ответах, где есть тренер/сотрудник с ценой, добавлены поля **`coach_price_1_2`** и **`coach_price_3_4`** (number или null).
- При **создании** и **редактировании** сотрудника (POST/PATCH staff) эти поля можно передавать в body.
- В CRM: в формах создания/редактирования тренера добавить два поля «Цена за час (1–2 игрока)» и «Цена за час (3–4 игрока)»; в превью брони/лобби использовать цену в зависимости от числа игроков (1–2 или 3–4).
