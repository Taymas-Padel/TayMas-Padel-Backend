# Для фронта: отмена брони, возврат часов, статусы броней

## Статусы брони (booking.status)

На бэкенде у брони четыре статуса. Фронт должен их все отображать и учитывать в логике (кнопки, фильтры, подписи).

| Значение (API) | Описание для пользователя | Когда показывать кнопку «Отменить» |
|----------------|---------------------------|------------------------------------|
| `PENDING`      | Ожидает оплаты            | Да (если разрешено по правилам отмены) |
| `CONFIRMED`    | Подтверждено              | Да (если разрешено по правилам отмены) |
| `CANCELED`     | Отменено                  | Нет |
| `COMPLETED`    | Завершено                 | Нет |

- **Авто-завершение:** бэкенд сам переводит брони в `COMPLETED`, когда наступает время окончания (`end_time`). Это происходит при загрузке списка/истории броней (GET `/api/bookings/`, GET `/api/bookings/history/`). Отдельный запрос или крон со стороны фронта не нужны.
- **Что сделать на фронте:** для статуса `COMPLETED` показывать подпись «Завершено» (или аналог), не показывать кнопку отмены; в фильтрах/вкладках при необходимости учитывать «Завершённые» отдельно от «Предстоящих» и «Отменённых».

## Что изменилось на бэкенде (отмена и абонемент)

1. **Возврат часов при отмене брони**  
   При отмене брони (`POST /api/bookings/<id>/cancel/`) бэкенд теперь **автоматически возвращает часы на абонемент**, если бронь была оплачена часами (PADEL-абонемент). Дополнительных запросов от фронта не нужно — после успешного ответа отмены часы уже зачислены.

2. **Новое поле в ответах по броням**  
   В ответах GET по брони (список броней, детали брони, история и т.п.) у объекта брони появилось поле:
   - **`membership_used`** — `number | null`  
     - Если бронь была оплачена часами абонемента — id этого абонемента (число).  
     - Если оплата деньгами или иначе — `null`.

## Как использовать на фронте

- **Отображение:**  
  Если у брони `membership_used !== null`, можно показывать метку вида «Оплачено по абонементу» / «Часы абонемента».

- **Отмена:**  
  Вызов `POST /api/bookings/<id>/cancel/` без изменений. После успешного ответа:
  - статус брони на бэкенде станет отменённым;
  - если у брони был `membership_used`, часы уже возвращены на этот абонемент.

- **Обновление остатка часов:**  
  После отмены брони, которая была «по абонементу», имеет смысл обновить данные пользователя (например, заново запросить `GET /api/auth/home/` или `GET /api/memberships/my/`), чтобы показать актуальный остаток часов.

## Ответ отмены брони

Успешный ответ `POST /api/bookings/<id>/cancel/`:

```json
{
  "status": "Бронирование отменено.",
  "hours_returned": 1.0
}
```

- **hours_returned** — число (float): сколько часов возвращено на абонемент. `0` — если бронь была оплачена не часами (деньгами) или создана до введения поля `membership_used` (такие брони не возвращают часы).

## Когда часы не возвращаются

- Бронь была оплачена деньгами (не абонементом).
- Бронь создана до обновления бэкенда (у таких броней `membership_used === null`).

## Кратко по API

| Что | Детали |
|-----|--------|
| **Отмена** | `POST /api/bookings/<id>/cancel/` — при оплате часами часы возвращаются автоматически. В ответе: `hours_returned` (число). |
| **Поле в брони** | `membership_used`: `number \| null` — id абонемента, с которого списаны часы, или `null`. |
| **Новые эндпоинты** | Нет. Меняется только поведение отмены и состав ответов по броням. |
