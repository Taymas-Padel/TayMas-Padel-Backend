# Бриф для фронта: участники при обычной брони — по желанию

## Что изменилось

Раньше при создании обычной брони (POST /api/bookings/create/) было обязательно указывать друзей под формат корта: 1×1 = ровно 1 друг, 2×2 = ровно 3 друга. Теперь **участники не обязательны** — можно забронировать корт **одним** (friends_ids может быть пустым массивом).

В **лобби** логика без изменений: состав и формат по-прежнему проверяются при наборе участников и создании брони из лобби.

---

## Бэкенд

- Валидация «ровно 1 друг для 1×1» / «ровно 3 друга для 2×2» в `CreateBookingSerializer` убрана.
- При `friends_ids: []` бронь создаётся на одного пользователя (владельца брони), участники не добавляются.
- Лимит по формату корта остаётся только как **максимум**: на корт 1×1 можно передать не больше 1 id в `friends_ids`, на 2×2 — не больше 3. Больше — по-прежнему ошибка валидации (если где-то проверяется) или просто лишние не учитываются. На бэкенде явной проверки «не больше 1/3» в сериализаторе нет — при желании можно добавить, но пустой массив и 1/3 друга уже ок.

---

## Что сделать на фронте (мобилка / веб-приложение)

1. **Форма «Забронировать корт» (обычная бронь)**  
   - Убрать проверку перед отправкой запроса вида: «если корт 1×1 — должно быть ровно 1 друг, иначе ровно 3 друзей».  
   - Разрешать отправку с **нулём выбранных друзей** (пустой массив `friends_ids`).

2. **Подсказка рядом с выбором друзей**  
   - Вместо текста типа «Корт 1×1: выберите ровно 1 друга» / «Корт 2×2: выберите ровно 3 друзей» использовать нейтральный вариант, например:  
     - «Участники (по желанию): корт 1×1 — макс. 1 друг»  
     - «Участники (по желанию): корт 2×2 — макс. 3 друзей»  
   - Чтобы было понятно: можно никого не выбирать, можно добавить до 1/3 друзей.

3. **Ограничение только по максимуму**  
   - Не давать выбирать больше 1 друга для корта 1×1 и больше 3 для 2×2 (как и раньше).  
   - Текст в предупреждении при переборе заменить с «нужно ровно N» на «можно добавить не более N друзей».

4. **Лобби**  
   - Ничего не менять: там по-прежнему нужен полный состав по формату (2 или 4 участника) перед созданием брони из лобби.

---

## Кратко

- **Обычная бронь:** друзья в брони — по желанию, можно бронировать одному; показывать «участники (по желанию)», лимит только макс. 1/3 по формату корта.  
- **Лобби:** логика «предложение кортов только по числу участников» и проверка состава не трогаются.
