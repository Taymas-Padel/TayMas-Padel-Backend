# API расписания тренера (для мобилки)

Два эндпоинта для отображения расписания тренера при входе в аккаунт тренера. Один даёт плоский список броней, второй — расписание по дням; **как именно показывать** (сетка, календарь, список по дням, таймлайн дня) решает приложение, данных из API достаточно для любого варианта.

---

## 1. Список броней (как есть)

**GET** `/api/bookings/coach/schedule/?from=YYYY-MM-DD&to=YYYY-MM-DD`

- **Права:** авторизованный пользователь с ролью `COACH_PADEL`, `COACH_FITNESS` или `ADMIN`.
- **Параметры (query):**
  - `from` — начало периода (дата, опционально). По умолчанию — сегодня.
  - `to` — конец периода (дата, опционально). По умолчанию — через 14 дней от `from`.

**Ответ:** массив броней (плоский список).

```json
[
  {
    "id": 42,
    "court": 1,
    "court_name": "Корт 1",
    "user": 10,
    "client_name": "Иван Иванов",
    "players_for_match": [{"id": 10, "name": "Иван Иванов"}, {"id": 11, "name": "Петр Петров"}],
    "start_time": "2026-03-15T10:00:00+06:00",
    "end_time": "2026-03-15T11:30:00+06:00",
    "duration_hours": "1.50",
    "price": "15000.00",
    "status": "CONFIRMED",
    "is_paid": true,
    "coach": 5,
    "coach_name": "Алексей Тренеров",
    "services": [],
    "participants_names": ["petr"],
    "membership_used": false,
    "created_at": "2026-03-10T12:00:00+06:00"
  }
]
```

Удобно для экрана «Предстоящие брони» (просто список).

## 2. Сетка расписания по дням (для календаря/недели)

**GET** `/api/bookings/coach/schedule/grid/?from=YYYY-MM-DD&to=YYYY-MM-DD`

- **Права:** те же — тренер или админ.
- **Параметры (query):**
  - `from` — начало периода (дата, опционально). По умолчанию — сегодня.
  - `to` — конец периода (дата, опционально). По умолчанию — через 14 дней от `from`.

**Ответ:** объект с диапазоном и расписанием по дням. Для **каждого дня** в диапазоне есть ключ (дата `YYYY-MM-DD`), значение — массив броней в этот день (если броней нет — пустой массив). Так можно рисовать сетку/календарь и не гадать, какие дни «пустые».

### Как показывать расписание в приложении (один API — разные экраны)

Эндпоинт **`/coach/schedule/grid/`** отдаёт для каждого дня массив броней. Как именно рисовать экран — на ваше усмотрение. Варианты:

| Вариант | Как сделать |
|--------|-------------|
| **Список по дням** | Проходите по `schedule` по порядку дат. Для каждой даты: заголовок «Среда, 12 марта» → под ним карточки броней (время, корт, клиент). Пустой день — блок «Нет занятий». |
| **Календарь (месяц)** | Сетка 7×5 (недели × дни). В ячейку даты выводите число и, например, точку/цифру количества броней. По тапу на день — открываете экран дня со списком `schedule[date]`. |
| **Неделя (сводка)** | Горизонтально 7 дней (Пн–Вс). Под каждым днём список броней этого дня (время + корт) или «—». Листание влево/вправо = смена недели, новый запрос с `from`/`to`. |
| **День (таймлайн)** | Пользователь выбрал дату → показываете только `schedule[эта_дата]`. Список по `start_time` или визуальная шкала времени (блоки 09:00–10:30, 14:00–15:30 и т.д.). |
| **Агенда (всё подряд)** | Можно даже не использовать «сетку»: взять `schedule` и вывести все дни подряд — «12 марта» → карточки, «13 марта» → карточки. Как один длинный список с заголовками дней. |

Итого: **один запрос** `schedule/grid/?from=&to=` даёт все данные; разметка (сетка, список, календарь, неделя, день) — на стороне приложения.

### Практические шаги

1. **Загрузка:** при открытии экрана расписания вызвать `GET /api/bookings/coach/schedule/grid/?from=...&to=...` (например, текущая неделя или две).
2. **Листание периодов:** при смене недели/месяца — тот же endpoint с новыми `from` и `to`.
3. **Детали брони:** по `id` из элемента в `schedule[date]` открыть детали через `GET /api/bookings/<id>/` при необходимости.

### Пример запроса (неделя)

```http
GET /api/bookings/coach/schedule/grid/?from=2026-03-12&to=2026-03-18
Authorization: Bearer <token>
```

---

## Общее

- В оба ответа попадают **все брони, где тренер назначен** (`booking.coach = текущий пользователь`), в том числе брони из лобби (когда в лобби выбран этот тренер).
- Время в ответах — в таймзоне сервера (обычно с смещением, например `+06:00`). Отображайте на клиенте в локальной таймзоне пользователя.
- Роли тренера: `COACH_PADEL`, `COACH_FITNESS`. `ADMIN` тоже видит расписание как «тренер».
